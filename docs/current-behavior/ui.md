## Current Contract

- Visible button targets are interactive across their full visual surface, not only across their text, emoji, or icon.
- Native SwiftUI button styles may own their native hit areas.
- Custom or plain buttons must make the intended button surface fill the target and define a matching `contentShape`.
- Routina glass-backed cards, pills, and panels provide rounded hit shapes through their shared visual modifiers so new glass-backed buttons inherit the rule by default.
- Mac Home uses a root-owned SwiftUI top toolbar chrome aligned to the macOS traffic-light titlebar band in normal windows. The Home split content is laid out underneath that chrome instead of being covered by it, and the toolbar is one titlebar-height row rather than a search row plus a second command row. In fullscreen, Home observes only the window fullscreen state and integrates the native titlebar/traffic-light reserve into that existing top toolbar chrome; it does not add a separate blank band above the toolbar, observe pointer-revealed titlebar state, or add/remove top padding while the pointer moves. The sidebar toggle sits in the lower-left titlebar pocket below the native traffic lights, while the rest of Routina's toolbar controls start after the toolbar's leading traffic-light-safe region, and the sidebar and main content remain below the custom toolbar row. Fullscreen Home disables full-size transparent titlebar content and avoids ignoring the top safe area so sidebar or split-view surfaces cannot show a rounded top edge behind the traffic lights; normal non-fullscreen windows keep full-size transparent titlebar content for their existing custom toolbar alignment. A leading sidebar control in the toolbar row explicitly collapses or expands the left Home sidebar with the normal sidebar animation, and it stays available after collapse because it lives in the top toolbar rather than inside the sidebar. While visible, the Home sidebar is clamped to its configured minimum and maximum widths so dragging the divider cannot make it consume the main workspace, and restoring the sidebar does not grow the Home window, animate right-side companion panes off the window edge, or make companion/task-detail surfaces brighten, duplicate, or ghost during sidebar visibility or task-detail expand/minimize transitions. The Done status badge is hidden by default and can be restored with Settings -> Appearance -> Toolbar -> `Show Done count in toolbar`; the SwiftUI-owned search pill sits in the same row, and the compact Tasks/Stats/Add mode strip renders immediately to the right of search. Places when enabled, the optional Stats/Adventure progress picker, and the board inspector button also live in this row when available. The SwiftUI-owned search pill is compact when idle and animates to its focused width when the user is searching or creating. Its rounded background, stroke, magnifying-glass icon, placeholder, clear button, create hint, and `Esc` keycap animate together and stay bounded inside the pill; typing and first-responder ownership are handled by a transparent AppKit text editor inside that shell. Focused search can be dismissed without clearing the query by clicking outside the visible search pill, pressing `Esc`, or using the toolbar search `Esc` keycap button, while the in-field clear button clears the query. It uses the shared Home search text so task lists, Timeline-style lists, the Planner Timeline segment's full Timeline surface, and task-backed Planner Calendar items can be searched from the top of the window. The field placeholder names both search and task creation, and a compact `Return` / `Create task` hint appears when the focused non-empty query has no matching task or timeline result. When that focused create-eligible query also contains quick-add syntax such as due words, recurrence, tags, places, priority, or duration, a flat parser preview attaches below the toolbar chrome showing the cleaned task title and detected details without duplicating the `Return` / `Create task` hint. The configurable Mac Quick Add shortcut focuses this toolbar field. Pressing Return with a non-empty query creates a task through the shared Quick Add parser only when the query has no task or timeline result; otherwise the field remains a live search. Successful toolbar creation clears search and shows the created-task toast, subscription limits open the paywall, and creation errors use a Home alert. The visible mode strip omits Timeline because Planner `Timeline` is the timeline review route, and the former top toolbar filter icon is removed. The Planner Calendar header shows Focus beside the filter button; the Focus control keeps task-picker duration starts, active plan-focus controls, and active non-plan focus badges local to Calendar mode. The Planner header filter button opens Home-level filters in a right-side `Both` / `Task List` / `Timeline` / `Calendar` companion pane; shared tag and importance/urgency filters live under `Both`, Timeline has no range filter, and Calendar owns Planner layer visibility filters. In Planner Timeline mode, `Go to date` remains visible beside filters, uses a single selected date instead of the Calendar visible range, and opens the right-side Planner date picker without scoping the Timeline list; selecting a date scrolls to the matching visible Timeline day when that day exists. Task and compatibility Timeline sidebars do not render duplicate search fields or filter icon buttons for that same toolbar area, though they may show active-filter summaries and clear actions. The toolbar field restores first responder after search updates so typing stays focused there, but it does not reclaim focus after the user moves into another text editor such as task comments or notes.
- In Planner Timeline mode, the Planner header filter button reflects active Timeline filters and opens the `Timeline` scope of the companion filter pane.
- The compact Mac Home toolbar mode strip omits Settings. The underlying Settings mode and routes remain available outside that toolbar strip.
- On Mac, task detail presentation follows the active workspace. Mac Home opens at 1440 x 760 and cannot resize below 1440 x 720, keeping enough horizontal room for the expanded left sidebar, a Day-capable Planner surface, and a right-side companion pane during sidebar animation. When Planner is active, single-click task-list selection and Planner task selection open a right-side companion detail pane beside the calendar when the detail area can fit the fixed companion pane plus a Day-capable Planner surface. At tight widths, the Planner calendar can adapt down to a compact Day layout to make room for the pane, and the Planner header hides the Day/3 Days/Week picker unless the companion layout has roomy inspector width. Hiding that picker does not by itself make Calendar/Timeline icon-only or shorten the date/range button; if the header narrows further, Calendar/Timeline text hides before the date/range button switches to compact width. The task-detail companion pane does not render a separate `Task Details` toolbar strip; its fullscreen and close controls live in the compact task header action cluster beside the completion action. Home filter companion panes keep their pane header controls, and fullscreen views opened from companion panes can minimize back to the previous pane layout. Those panes are mutually exclusive with each other, Planner's internal right sidebar, and the board inspector. Full Details close still returns to Planner and clears the task-detail pane. Double-clicking a Mac task-list row opens the full Details surface.
- Mac task-detail actions render as one explicitly spaced top-right control cluster inside the task detail header card rather than in the app toolbar, so task-specific actions stay owned by the selected task and do not compete with toolbar search. `Done` is the only prominent task action, using a fixed-height custom rounded-rectangle label so its short text does not collapse into a circle. Full Details keeps routine Pause/Resume and one-off `Cancel todo` as secondary text actions, with `Cancel todo` using a non-close icon instead of an `xmark.circle` near the close control; link, sharing, edit, minimize, and close render as equal-size icon controls, with close remaining the single rightmost `xmark` for leaving Full Details. The right-side task-detail companion pane uses the compact variant: completion action plus fullscreen and close only, omitting Pause/Resume, `Cancel todo`, link/share, and edit controls. Mac task form creation and inline editing keep Cancel/Save inside the Identity section rather than in the app toolbar.
